# DocuGen AI — 智能文档格式化平台

## 使用指南

### 一、为什么需要这个工具？

作为售前、项目经理或技术人员，你一定遇到过这样的烦恼：

- 网上收集的资料粘贴到 Word 后**格式乱七八糟**，手动用格式刷一段一段刷，费时费力
- 客户发来的聊天记录、邮件内容需要整理成正式文档，**人工排版耗时巨大**
- 标书技术部分几十页内容，格式不统一，**逐页调整标题、表格、缩进**极其痛苦
- 复制的 Markdown、网页文本、PDF 内容，粘贴后**丢失结构**，需要重新排版

**DocuGen AI** 就是为了解决这些问题而生 —— **AI 一键格式化，告别手动格式刷。**

不管是网上收集的资料、Markdown 文本、复制粘贴的内容、原本格式不规整的 Word 文档，还是标书的技术部分，都可以通过本工具一键转写为**格式规范、结构清晰**的标准文档输出。

### 三、核心功能

#### 1. AI 格式化

将任意格式的文本一键转换为标准 Markdown 格式文档。

**使用方法**：在编辑区粘贴内容 → 点击侧边栏「AI 格式化」按钮

**支持的输入格式**：
- 网上收集的各种资料（网页复制、论坛帖子等）
- 微信 / 钉钉聊天记录
- 会议记录、零散笔记
- Markdown 文档
- 格式不规整的文本内容
- 任意非结构化文本

> 长文本（超过 4000 字符）会自动分段处理，确保不丢失内容。

#### 2. 智能表格

AI 自动识别文本中的表格类数据，将其转换为标准表格格式。

**使用方法**：粘贴含有表格数据的内容 → 点击「智能表格」

适用场景：从 PDF 复制的表格数据、Excel 粘贴到聊天中的表格、列表形式的数据等。

#### 3. 标书转写（实验功能）

专为**标书技术部分**等长文档设计的格式化功能。使用 Gemini 长上下文模型，支持最大 **50MB** 文件上传，自动分段处理并保持全文格式一致。

**使用方法**：点击「导入标书」上传文件 → 点击「标书转写」

**特点**：
- 强制使用 Gemini 长上下文模型，减少分段次数，格式更一致
- 单段处理字符数是普通模式的 3 倍（12000 字符），上下文更完整
- 严格禁止修改原文内容，只调整格式和结构
- 支持 .docx / .pdf / .txt 格式

> **注意**：这是实验功能，因上下文较长可能产生幻觉，转写后请自行人工校验。

#### 4. 导入文件

支持直接上传文件，自动提取文本内容到编辑区。

**支持格式**：

| 格式 | 说明 |
|------|------|
| .docx | Word 文档（自动提取段落和表格） |
| .xlsx | Excel 表格（每个 Sheet 转为 Markdown 表格） |
| .txt / .md | 纯文本和 Markdown |
| .csv / .json / .xml | 结构化数据文件 |
| .html / .log | 网页和日志文件 |
| .pdf | PDF 文档（服务端解析） |

**使用方法**：点击侧边栏「导入文件」按钮 → 选择文件

> 普通文件限制 10MB，标书模式支持最大 50MB。

#### 5. 导出文档

将编辑区的内容导出为正式文档格式：

- **导出 DOCX**：生成 Word 文档，支持选择字体（微软雅黑 / 仿宋 GB2312 / 宋体）
- **导出 Excel**：将文档中的表格提取为 Excel，每个表格一个 Sheet

**使用方法**：点击侧边栏「导出 DOCX」或「导出 Excel」

#### 6. 历史文档

每次导出文档时会自动保存一份副本，方便日后查阅和复用。

**使用方法**：点击侧边栏「历史文档」→ 在右侧抽屉中浏览 → 点击加载

#### 7. 模型选择

支持多个 AI 模型，可根据需要在侧边栏切换：

| 模型 | 特点 |
|------|------|
| DeepSeek V3.2 | 推荐，中文理解力强 |
| Gemini 2.5 Flash | Google 模型，速度快，长上下文 |
| Kimi K2.5 | 备选 |

### 四、典型使用场景

#### 场景 1：网上资料整理成文档

1. 从网页、论坛、公众号等复制收集到的资料
2. 粘贴到编辑区 → 点击「AI 格式化」
3. AI 自动整理为标题层次分明、段落清晰的规范文档
4. 导出 DOCX，直接可用

> **告别格式刷**：以前需要花 30 分钟手动调格式，现在 10 秒搞定。

#### 场景 2：聊天记录整理格式

1. 复制微信/钉钉中的工作讨论记录
2. 粘贴到编辑区 → 点击「AI 格式化」
3. AI 自动整理排版格式（如统一时间格式、对齐发言人、分段落），原始内容完整保留
4. 导出 DOCX 存档

#### 场景 3：格式混乱的 Word 重新排版

1. 点击「导入文件」上传格式不规整的 .docx 文件
2. 编辑区自动显示提取的文本内容
3. 点击「AI 格式化」一键重新排版
4. 导出为格式规范的新 DOCX

#### 场景 4：标书技术部分格式化

1. 点击「导入标书」上传标书 Word/PDF 文件（最大 50MB）
2. 编辑区显示提取的标书内容
3. 点击「标书转写」，AI 使用长上下文模型进行格式化
4. **人工校验**转写结果（实验功能，务必检查）
5. 导出为 DOCX

#### 场景 5：会议笔记排版整理

1. 将会议中的零散笔记粘贴到编辑区
2. 点击「AI 格式化」
3. AI 自动识别笔记结构、整理标题层级和列表格式（不会增删内容，只调整排版）
4. 点击「导出 DOCX」发送给参会人员

> **注意**：AI 只做格式整理，不会自动补充总结或待办事项。如需会议纪要模板，建议先按"议题—讨论—结论"格式记录，再用 AI 格式化排版。

#### 场景 6：Excel 数据整理

1. 导入 .xlsx 文件（或粘贴从 Excel 复制的表格数据）
2. 点击「智能表格」让 AI 优化表格结构
3. 点击「导出 Excel」下载整理后的表格

### 六、常见问题

**Q：注册时提示 API Key 无效？**
A：请确认从 [aiapi.colasoft.cn](http://aiapi.colasoft.cn) 获取了有效的 Key，并检查网络是否正常。如需帮助请联系管理员。

**Q：AI 格式化后内容丢失？**
A：不会丢失。超过 4000 字符的长文本会自动分段处理，所有内容均会保留。

**Q：标书转写后内容有误？**
A：标书转写为实验功能，因文本较长，AI 可能产生幻觉。请务必人工校验转写结果，确认无内容遗漏或修改。

**Q：导出的 Word 字体显示异常？**
A：如果选择了「仿宋 GB2312」字体，需要在本机安装该字体。侧边栏字体选择处有下载链接。

**Q：可以上传大文件吗？**
A：普通模式限制 10MB，标书模式支持最大 50MB。
